table-layout

table-layout används för att ställa in hur webbläsaren ska rita en tabell. Inställningen kan göra ritningen av tabellen snabbare genom att använda inställningen fixed. fixed gör så att webbläsaren tvingas att rendera alla kolumner med samma bredd som bredden hos den första radens kolumner, något som gör renderingen snabbare. auto kan också användas och berättar då för webbläsaren att "rendera som du vill". form och CSS

Formulär kan modifieras på (förstås) många olika sätt med CSS. Ett bra sätt att få sina formulär att se bättre ut är att ställa in fasta storlekar för alla label-element som associeras med input-element. På detta sätt kan man få en rak linje med beskrivande text och dess inputfält. Nedan ser vi ett exempel på denna inställning i ett basic formulär.

Personinfo

Alla labels på alla formulär som visas på den här sidan har fått samma pixelstorlek: 

        form .label {
            display: inline-block;
            width: 100px;
            text-align: right;
            padding: 10px;
        }

Det är också viktigt att ställa display-inställningen till inline-block för att ens kunna sätta bredden på <input>-taggen.

Pseudo-klasser

Det går givetvis också bra med dessa inputfält att använda pseudo-klasser, nedan kan vi se hur fältet ändrar färg när det väljs gentemot hur det ser ut när det är inaktivt.

Personinfo

För att stylea input-fält på detta sätt måste pseudo-klassen :focus användas. I detta fall så sätter vi då tillbaka färgen till vit när fältet markeras, så här: 

        form.color input:focus {
            background-color: white;
        }

Grupperingar med <fieldset>

Ett fieldset är ett bra sätt att gruppera input-element och ger bra möjlighet för CSS-formattering. Nedan ser vi hur bakgrunden har ändrats samt att kanterna jämnats ut. Här har vi även en lite mer stylead submit-knapp.

Personinfo

Följade CSS användes för att stylea fieldset: 

        form.fieldset fieldset {
            border-radius: 25px;
            background-color: beige;
        }
