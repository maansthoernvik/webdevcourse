Här kommer exempel på de mest centrala delarna av alla HTML-dokument!

  • html taggen innesluter hela dokumentets innehåll, därför sätts tillexempel lang=sv på denna tagg (som nämnt på sidan innan) för att märka vilket språk hela sidan är skriven på.
  • I head så skrivs mer information om sidan in, här kan saker såsom stylesheet, javascriptfiler som används, meta-information och sidans titel läggas till.
  • title-taggen som läggs in i sidans head-tagg kommer att visas överst i webbläsarfönstret (på tabben om man tillexempel använder sig av Google Chrome).
  • body-taggen är sidans faktiska sidans innehåll, antingen det som ska läsas, ses eller höras ska läggas till här.
  • Meta-taggar innehåller information om data på sidan. Meta-taggar kan ha ett name-attribut beskriver det som skrivits in efter name i ett content-attribut, exempel på name-attribut till meta-taggen är:
    • author beskriver sidans författare. Exempel på content attribut för denna kan vara Måns Thörnvik, eftersom att jag är författaren.
    • description beskriver sidans innehåll. Exempel på content attribut kan vara Distanskurs i webbutveckling.
    • keywords beskriver vad sidan kan kopplas till för nyckelord. content kan vara HTML, CSS, JavaScript, JS mm. Dessa kan läggas till som en kommaseparerad text till content såsom nedan.
    • robots låter dig kontrollera hur sidan ska visas i sökresultat. Tillexempel så kan content sättas till 'noindex' för att inte visa sidan i sökresultat.
    • generator specificerar en av de mjukvaror som använts för att generera html-dokumentet i fråga. I detta fall N/A sedan sidan skrivits för hand och inte genererats.
    • Så här skulle alltså denna sidas <meta>-taggar utformas: 
                        <meta name="author" content="Måns Thörnvik">
                  <meta name="description" content="En introduktion till HTML5-standarden">
                  <meta name="keywords" content="HTML5,html,webb">
    http-equiv användes tidigare som attribut på en meta-tagg(innan HTML5) för att sätta character-encoding som gicks igenom på förra sidan. Numera så kan denna också användas för att sätta default-stylesheet och ett refresh attribut. Refresh attributet måste sedan kompletteras med ett tidsintervall i content-attributet och sidan kommer då att laddas om efter den spesade tiden. Denna refresh bör dock inte användas eftersom att den tar kontrollen av sidan från användaren. Exempel på refresh finns här!
