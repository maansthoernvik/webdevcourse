span är en tagg som kan vara användbar för att stylea text inuti ett textflöde. div är ett annat element som precis som span inte egentligen gör någonting men div skiljer sig i det att det elementet bryter den nuvarande raden. span är nedan satt runt texten "span span span" och som det syns så påverkar den ingenting, dock så kan man som sagt fritt stylea den texten inuti span taggen genom att koppla den till ett id eller klass eller en generell CSS-regel som appliceras på alla span-taggar. För att koppla ett html-element till ett id eller en klass använder man attributet id eller class på den tagg som man vill sätta ett id på. Så här:
<span id="span_id" class="span_class">Min span text</span>

Här är en massa text som löper på... här kommer ett span span span men det påverkar inte formattering eller någonting annat, så länge man inte vill att det ska det och kopplar in ett stylesheet.

Bilder sätts in i ett html-dokument med hjälp av img-taggen. Bilder behöver källan från vart bilden ska hämtas vilket spesas med src-attributet på img-taggen. Det går även att lägga till ett alt-attribut för bilder som är en beskrivande text om bilden som visas om bilden inte kan hämtas av någon anledning. Bilden på bilen nedan har spesats på följande sätt: 

      <img src="ugly_car.jpg" alt="Ugly car">

Lägg märke till att img-taggen inte behöver en sluttagg </img> såsom många andra taggar gör!

a, anchor tags kan användas på flera olika sätt. De kan användas för navigering (såsom denna genomgång strukturerats med länkar mellan sidor), för att skicka e-post eller ladda ner material och de kan även se ut på många olika sätt. För att peka på en plats som a taggen ska ta användaren till så används attributet href="plats att gå till". Den spesade platsen kan vara ett annat html-dokument, en mailadress eller en serveradress, mm.

Bilden ovan är en länk, a taggen kan innehålla mängder av material och alla element som läggs inuti den blir klickbart. Bilden ovan är ett element mellan start och sluttaggen för en anchor-tag.

En annan viktig del av anchor-tags är att destinationen de pekar mot kan vara relativ platsen du just nu är på eller fast. En relativ destination spesas på följande sätt:
destination_jag_vill_gå_till.html
medans en absolut länk spesar hela sökvägen, så här:
http://www.minhemsida.se/destination_jag_vill_gå_till.html

För att skapa en länk som kan användas för att skicka ett mail så måste man lägga till ytterligare information i anchor-taggens href attribut så att den vet att den ska skapa ett mail när den klickats på. Detta görs genom att skriva:
href="mailto:mans@gmail.com?Subject=ämne"
Subjectdelen kan läggas till om man även till att ämnet automatiskt ska komma med den skapade e-posten. Den här länken kommer att öppna den e-postklient som är default på användarens datum, skapa ett nytt utkast samt lägga till "hello" i ämnesraden. Klicka här för att skapa ett nytt e-postmeddelande!

Anchortags kan inte bara användas för att länka till något utanför den nuvarande sidan, klicka här för att gå tillbaka till sidans topp. Genom att lämna en tom attribut för href så länkar denna tillbaka till den nuvarande sidans topp. Man kan dock även sätta idn hos olika html-element som tidigare nämnt. Om man sedan skriver '#' efterföljt av elementets id som href navigerar nu den a taggen till det givna id-numret. Klicka här för att gå till beskrivningen av 'span' och se hur vi kan navigera till en specifik plats på en webbsida. Notera att addressfältet i din webbläsare ändras till "nuvarande plats på sidan/#span" för att indikera att idt "span" är dit vi navigerat efter att länken har klickats.

Slutligen så kan man även med hjälp av anchor tags även länka till andra protokoll än http, som är standard för webbsidor. Behöver man länka till en ftp-server för att exempelvis hämta en fil så måste man speficicera vilket protokoll som följs hos destinationen genom att skriva:
href="ftp://adressen till servern" för href-attributet för den anchor tag som ska kontakta ftp-servern.
