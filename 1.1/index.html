  • HTML-dokument byggs upp med hjälp av taggar. Taggar kan introducera antingen layoutinställningar och /eller semantiska meningar med en bit text, ett område på skärmen eller ett helt dokument. Hela dokumentet innesluts av html-taggen, sedan en head sektion som innehåller deklarationer för stylesheets och inkluderade javascript-filer och till sist body där sidans material/text/bilder/video skrivs in.

    För att innan vi går vidare ge en kort beskrivning om hur HTML fungerar så ger jag här en kort beskrivning för att ni ska kunna hänga med på senare sidor:
    <tagg attribut="värde för attribut"> Element eller text inom taggen </slut-tagg>
    Ovan ser ni hur HTML fungerar. En tagg har en början, ett innehåll och ett slut. Start-taggen kan ha attribut som påverkar utseende och/eller funktionalitet. Notera dock att alla taggar INTE har en slut- tagg, till exempel taggen för radbrytningar <br>. Slut-taggar har alltid ett snedstreck '/' för att indikera att de är just sluttaggar. Nedan ser du hur taggar kan innesluta andra tagget och därigenom skapa HTML-dokumentets hierarki: 

              <html>
          <head>
            <title>Sidans titel</title>

            <meta charset="UTF-8">
            <meta ...>

            <script src="script.js">
            <link rel="stylesheet" href="styles.css">
          </head>
          <body>
            Sidans innehåll läggs till här med taggar, text, video och annat material!
          </body>
          </html>
  • !DOCTYPE indikerar vilken standard som dokumentet följer. <!DOCTYPE html> som sätts in i den översta delen av HTML-dokumentet betyder att dokumentet följer HTML5-standarden.
  • charset="UTF-8" indikerar att UTF-8 används för att konvertera dokumentet när det sparas och när det sedan laddas upp igen. UTF-8 är vanligast idag för Engelska webbsidor. Denna inställning spesas i dokumentets head-sektion som ett eget element med hjälp av en <meta>-tagg. Ställ in detta genom att sätta attributet charset="UTF-8" på meta-taggen.
  • lang=sv som är satt som attribut på html-taggen är en konvention som bör användas för att indikera vilket språk sidan är skriven på. Det påverkar inte sidan mer än att till exempel sökmotorer kan använda informationen.
  • Valideringsverktyg kan användas för att bekräfta att ett HTML-dokument är korrekt utformat. Det kan upptäcka saker såsom: saknad titel, saknade sluttaggar för olika element och taggar som saknar obligatoriska attribut. W3C som utvecklar HTML5-standarden tillhandahåller ett webbverktyg som man kan använda för att validera sina HTML-dokument för att se att de följer standarden. Detta verktyg kan hittas på den här länken: W3C HTML5-validator
